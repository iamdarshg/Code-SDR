# Code-SDR V2 PCB Routing Strategy

## Current Status

- **PCB:** Code-SDR-V2.kicad_pcb
- **Footprints:** 344
- **Nets:** 1,514
- **Current routing:** 0 tracks, 0 vias (COMPLETELY UNROUTED)
- **Layers:** 4-layer board (F.Cu, In1, B.Cu, plus In2 referenced in design)
- **Complexity:** RF receiver with 10 GHz capability, Ethernet, FPGA, ADC, LO chains

## Design Constraints

### RF Critical Nets (Must stay on F.Cu, zero vias)
- 915 MHz RF nets
- GNSS RF nets  
- Mixer LO lines
- IF chain differential pairs
- LO chain critical paths
- Balun connections

### Differential Pair Skew Limits
- **USB D+/D-:** 1.0 mm max
- **Ethernet MDI pairs:** 1.0 mm max each
- **LO pairs:** 0.75 mm max
- **IF pairs:** 0.50-0.75 mm max
- **ADC pairs:** 0.50 mm max

### Layer Rules
- **F.Cu:** Main signal routing layer for high-speed and RF
- **In1:** Solid GND plane (NO signal routing allowed)
- **B.Cu:** Secondary signal routing layer
- **In2:** Power plane layer

### Via Restrictions
- **RF critical nets:** ZERO vias allowed
- **Ground vias:** Only in GND regions
- **Signal vias:** Restricted to non-critical regions with proper clearance

## Routing Strategy

### Phase 1: Establish Baseline
✓ Verify schematic (ERC pass)
✓ Confirm PCB structure (footprints, zones, power planes)
✓ Create baseline metrics (before routing)
- [ ] Export clean DSN without any existing routing

### Phase 2: Constrained Routing
The board is too complex for unconstrained autorouting. Use staged approach:

1. **Route non-critical local connections**
   - Power domain local returns
   - Supply bypass paths
   - Short GPIO signals
   - Target: ~200 connections

2. **Route bus signals (second stage)**
   - ADC data bus (ADC_D0-D9)
   - RGMII signals (TX/RX data, controls)
   - FPGA configuration bus
   - Target: ~100 connections

3. **Route differential pairs (third stage)**
   - USB D+/D- as matched pair
   - Ethernet MDI as matched pairs (A, B, C, D)
   - With skew constraints verified after each pair

4. **Route LO and IF chains (final stage)**
   - LMX2592 LO outputs to mixers
   - Fixed 2.410 GHz LO
   - AD9215 sample clock
   - IF chain differentials
   - With F.Cu-only enforcement

5. **RF front-end (last)**
   - Only using deterministic manual routing for critical RF
   - Balun connections
   - Switch paths
   - Pre-filter networks

### Phase 3: Validation & Iteration
- Run DRC after each phase
- Verify skew after differential pairs
- Check layer compliance for RF nets
- Ensure no signal on In1 plane
- Validate connectivity

## Files and Workflow

### Key Scripts
- `tools/generate_pcb.py` - Regenerate PCB layout from design model
- `tools/export_route_checkpoint.py` - Export DSN for routing
- `tools/import_route.py` - Import FreeRouting SES result
- `tools/add_rf_via_fences.py` - Add ground via fences around RF routes
- `tools/validate_route.py` - Validate final routed design

### Validation Requirements

**Before committing any routed result:**

1. [ ] PCB opens successfully in KiCad
2. [ ] All footprints present and accounted for
3. [ ] All nets defined
4. [ ] Nonzero tracks and vias (not a lost route)
5. [ ] Track count improved from source checkpoint
6. [ ] No unexpected copper loss
7. [ ] DRC: zero unconnected items
8. [ ] DRC: zero error-severity violations
9. [ ] Layout audit: all RF constraints pass
10. [ ] Layout audit: all skew constraints pass
11. [ ] Layout audit: solid In1 GND plane preserved
12. [ ] Manual inspection of critical RF paths

## Notes

- The DSN export intentionally omits GND from routing (solid In1 plane)
- RF routes must not be regenerated by `generate_pcb.py` after import
- Schematic hierarchy paths must be refreshed after SES import
- Via fences around RF routes are added as a post-import step
- FreeRouting is used with `-mt 0` to disable optimization passes for stability
